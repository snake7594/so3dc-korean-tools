## FIS 포맷

FIS는 tri-Ace GS/GIF DMA 텍스처 패킷이다. CLUT 바이트 크기 `u32@0x20`, CLUT
RGBA8888 `0x100`(CSM1 순서: 인덱스 비트 3↔4 스왑), PS2 알파 `A_out=min(255,A×2)`.
픽셀 클래스 5종: PSMT4+CLUT(4bpp), PSMT8+CLUT(8bpp), PSMCT32+CLUT(=8bpp를
32bpp로 위장, 실제 크기 2×TRXREG, PS2 8비트 언스위즐), PSMCT32 직접(32bpp),
PSMCT24 직접(24bpp).

## 검증 항목

1. 출력 ISO 크기 `4,689,854,464`, 원본 ISO SHA 고정
2. **텍스트 패치 완전 보존**: 8개 텍스처 멤버 영역 밖 전 영역이 `v1.0.0` ISO와
   바이트 동일(스트리밍 비교, 변경 0바이트). 변경 총 `797,558` 바이트가 전부
   8개 대상 영역 내부
3. 편집 전 원본 바이트 왕복 재현 증명(변경 없는 디코드를 재인코딩하면 원본
   멤버와 바이트 동일) — 픽셀 본문만 재작성, FIS 헤더·CLUT 바이트 보존
4. 각 멤버: 재압축 SLZ가 패치 바이트로 정확히 압축 해제, 멤버 footprint 불변,
   SLZ 체인 `next_rel` 정합, 원본 할당 내 수용(fail-closed)
5. 원본 대조로 8개 이미지 멤버가 텍스트 패치에서 미변경임을 확인(무충돌)
6. 출력 ISO에서 직접 디코딩한 텍스처가 패치 결과와 일치(라이브 확인)
7. xdelta 3.0.11 왕복 복원 ISO가 출력 ISO와 byte-identical

## 방법 요약

- 인덱스 텍스처(4/8bpp): 일본어 글자 색 인덱스와 국소 배경 인덱스를 표본화 →
  글자 픽셀을 배경 인덱스로 삭제 → 원본 글자 높이·기준선·여백에 맞춰 NanumSquare
  Neo로 한글 렌더 → 그레이스케일 커버리지를 원본 텍스트 색 램프 인덱스로 매핑.
  편집 영역 외 인덱스는 바이트 동일 유지.
- 사진 텍스처(24bpp): 국소 중앙값 대비로 캡션 마스크 → 조화(라플라스) 인페인트로
  글자 픽셀만 주변 사진으로 채움 → 한글을 원본 캡션 중심·높이에 맞춰 흰색+1px
  어두운 테두리로 렌더. 영문 부제·HUD·게이지(y≥28)는 바이트 동일.

## 알려진 제약

- 필살기 라벨(24bpp 사진)은 일본어를 지운 자리를 주변 사진으로 메운 뒤 한글을
  얹어, 원본보다 짧은 한글 좌우로 옅은 인페인트 흔적이 남을 수 있다(타이틀
  데모 전용). 텍스트 패치의 대사창 폭 잔여 제약은 `v1.0.0`과 동일.
- 실기 표시는 사용자 확인 항목.
