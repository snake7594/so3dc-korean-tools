# Star Ocean 3 DC Korean Tools

『스타 오션 3 Till the End of Time Director's Cut』 일본판 PS2 Disc 1의
숨겨진 tri-Ace 아카이브, SLZ 압축, `so3mclib` 메시지·폰트 형식을 분석하고
한국어 패치를 만드는 연구 도구입니다.

**Disc 1 + Disc 2 전체 한국어화** 릴리스입니다(`v1.2.0`). 게임 텍스트를 담은
mclib 컨테이너 전부(Disc 1 1,350개 / Disc 2 1,328개)를 대상으로 하며,
대사·화자, 메뉴, 아이템명·설명, 문장술·전투 스킬, 적 이름, 아이템
크리에이션, 시스템 다이얼로그를 포함합니다. 캐릭터 이름은 대사 참조뿐
아니라 파티·스테이터스 화면의 엔진 하드코딩 이름 테이블까지 한글화합니다.
이미지(텍스처)에 픽셀로 박혀 있던 일본어(초기 설정 설명문, 설치·뮤직 모드
안내, 디스크 교체·오류 메시지, 타이틀 데모 필살기 라벨)도 텍스트 패치와
동일한 NanumSquare Neo로 원본 글자 크기·위치·색에 맞춰 다시 그렸습니다.

두 디스크의 공통 텍스트(메뉴·아이템·전투 등)는 동일 번역을 공유하고,
디스크 2 고유 후반 스토리는 추가 번역했습니다.

> 이전 알파(`v0.1`~`v0.4`)는 부분 패치, `v1.0.0`~`v1.1.0`은 Disc 1
> 전용이었습니다. `v1.2.0`부터 두 디스크를 모두 다룹니다.

  아드레이·미라쥬 (대사 참조 + 엔진 하드코딩 이름 테이블 3벌 + 글로벌 폰트
  이름용 21개 슬롯 교체, 런타임 변환기는 무수정)

- 용량 한계 미교체 231개는 해당 아카이브가 원본 섹터 할당을 초과해 재압축·
  재배열로도 수용 불가한 소수 메시지이며, 결정론적으로 최소한만 제외

생성해 컨테이너별 로컬 아틀라스에 삽입합니다. 원본 ISO·아카이브·숨김 인덱스
크기는 불변으로 유지합니다.

## xdelta 적용

GitHub Releases에서 최신 릴리스 ZIP을 받아 풀고 디스크별로 적용합니다.

```powershell
xdelta3 -d -s "SO3_DC_Disc1_original.iso" `
  "SO3_DC_Disc1_Korean_Full_v1.2.0.xdelta" `
  "SO3_DC_Disc1_Korean_Full.iso"

xdelta3 -d -s "SO3_DC_Disc2_original.iso" `
  "SO3_DC_Disc2_Korean_Full_v1.2.0.xdelta" `
  "SO3_DC_Disc2_Korean_Full.iso"
```

- Disc 1 출력 ISO SHA-256:
  `1A735C6F3F3C7CF38DD6B23E1B983B3BF008721F23595528C538280768E12B2A`
- Disc 2 출력 ISO SHA-256:
  `4AE1507DFACF53144150FD9A2D2515F77770886EF94C3F1B982C6659D91AD224`

반드시 적법하게 소유한 원본 디스크 덤프를 사용하세요. 출력 ISO, xdelta,
릴리스 ZIP의 SHA-256은 릴리스 번들의 `SHA256SUMS.txt` 및 각 버전의
`docs/releases/vX.Y.Z-verification.md`에 기록합니다.

## 이미지(텍스처) 패치 (v1.1.0)

게임 텍스트 대부분은 mclib 글리프로 렌더되어 텍스트 패치가 처리하지만, 일부
UI는 일본어가 이미지(FIS 텍스처)에 픽셀로 그려져 있습니다. `v1.1.0`은 그중
일본어가 들어간 8개 텍스처를 한국어로 다시 그립니다.

- 초기 설정 항목 설명문(7줄), PlayStation BB Unit 설치 안내문, 뮤직 모드
  설명문, 디스크 교체·오류 시스템 메시지(DISC 1/2 교체·디스크 다름·읽기
  실패·확인 중·디스크 없음), 타이틀 데모 전투 필살기 라벨 4종
- 글꼴은 텍스트 패치와 동일한 NanumSquare Neo Bold. 원본 일본어의 글자
  크기·위치·색(CLUT 인덱스)에 맞춰 렌더링하고, 영문 그래픽·로고·버튼·캐릭터
  이미지 등 비대상은 원본 그대로 보존합니다.
- FIS는 tri-Ace GS/GIF DMA 텍스처 포맷(CLUT CSM1, PS2 알파, 4/8/24/32bpp,
  일부 8bpp-위장-32bpp 언스위즐)으로, 편집 전 원본 바이트 왕복 재현을 증명한
  뒤 캡션 영역만 다시 칠하고 SLZ로 재압축해 원본 할당 안에 수용합니다.
- 독립 검증: 8개 텍스처 멤버 영역 밖 4.69GB 전 영역이 텍스트 패치본과 바이트
  동일(텍스트 패치 완전 보존), 각 텍스처 FIS 헤더·CLUT 불변, SLZ 왕복 일치.

## 구현 개요

- 메시지 바이트코드 제어 opcode 36계열을 엔진 디스어셈블 3중 교차검증으로 전수
  확정(183,065/183,065 토큰화)해 전 메시지를 안전하게 재인코딩합니다.
- 범용 패처가 임의 PK1 패키지·행·SLZ 체인·PACK 테이블·glyph0 컨테이너를 다루며,
  실패 시 fail-closed로 중단합니다. 원본 ISO·아카이브·숨김 인덱스 크기는 절대
  바뀌지 않습니다.
- SLZ mode-2 최적 파스 압축기(그리디 대비 평균 +2.85%)와 비트맵 최근접 이웃
  재배열로 컨테이너를 원본 섹터 할당 안에 재수용합니다. 이름 테이블이 든 엔진
  코드 모듈은 SLZ mode-3 최적 인코더로 재작성합니다.
- 자동 픽셀 폭 게이트가 8a80 확대 배율을 반영해 각 라인 폭을 원본 예산 이내로
  맞춥니다.

## 정적 검증

독립 검증기는 패처 로직을 공유하지 않고 별도 경로로 결과를 다시 분석합니다.

- 원본·출력 ISO 크기·SHA-256, 폰트 SHA-256 고정
- 숨겨진 tri-Ace 인덱스(6,144 엔트리) 원시 바이트 + 디코딩 결과 불변
- 변경 범위가 대상 아카이브 1,006개 extent + 이름 테이블 4개 영역 안으로 한정,
  그 밖 4.69GB 전 영역 바이트 동일
- 한국어 화자·본문 293,025/293,025 발생 역디코딩 정확 일치
- 이벤트·페이지·마커·제어열 보존, 비대상 메시지·글리프 불변
- 각 컨테이너의 한글 글리프를 NanumSquare Neo로 재렌더해 아틀라스 일치
- 글로벌 폰트의 이름용 21개 슬롯만 교체, 나머지 271개·헤더 불변
- xdelta 3.0.11 왕복 복원 ISO가 출력 ISO와 byte-identical

빌드별 수치·해시는
[`docs/releases/v1.0.0-verification.md`](docs/releases/v1.0.0-verification.md)에
기록합니다.

## 알려진 제약

- 대사창 폭: 자동 폭 게이트 적용 후에도 축약 불가한 12개 메시지(19개 라인)가
  남습니다 — 런타임에 삽입되는 값(날짜·상점 수치·달성률·비밀번호 입력칸)이
  들어가는 칸과, 고유명사가 원본보다 1글자 미만(6~7px) 넓은 경우입니다.
- 유니코드 대응이 없는 장식 기호·아이콘 글리프는 원본 비트맵을 그대로 보존합니다.
- 정적 검증은 통과했고 실기 구동으로 초기 화면·대사 표시를 확인했으나, 152,695개
  전 메시지를 게임 진행 전 구간에서 하나하나 확인한 것은 아닙니다. 이상 표시가
  보이면 제보해 주세요.

## 법적 고지

이 프로젝트는 비공식 팬 연구 프로젝트이며 tri-Ace, Square Enix, Sony와 관계가
없습니다. ISO, 게임 실행 파일, 추출 게임 데이터, BIOS, PCSX2 세이브스테이트와
TTF 폰트 파일은 저장소나 릴리스에 포함하지 않습니다. 반드시 적법하게 소유한
원본 디스크 덤프를 사용하세요.
